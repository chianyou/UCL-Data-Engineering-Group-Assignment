Methodology

How data moves from sources to analytics and visualization.

Raw files are ingested from four public sources (NVD, CISA KEV, EPSS, and cvelistV5) and stored in a raw landing layer. Raw snapshots are then loaded into MongoDB for replayable processing.

Transformation is executed through PySpark and Python ETL to produce curated Parquet datasets, including the vulnerability mart, product bridge tables, star schema dimensions/facts, and aggregation summaries.

Curated Parquet is loaded into DuckDB. The dashboard consumes analytics through FastAPI endpoints (with CSV fallback), and product-level charts are calculated from the CVE-product bridge to preserve many-to-many CVE/product relationships.

Data can be loaded either from static CSV files under data/curated/transformation_summaries or from the FastAPI service using ?api_base=http://localhost:8001.
